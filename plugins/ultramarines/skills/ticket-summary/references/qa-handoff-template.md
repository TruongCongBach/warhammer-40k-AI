# QA Handoff Template

## Purpose

Use this template when the summary needs explicit validation guidance for QA.

## Template

```markdown
QA focus areas
- Primary flow to validate: ...
- Risky states to verify: ...
- Regression-prone areas: ...

Suggested test scenarios
- ...
- ...
- ...
```

## Guidance

- Call out the user-facing flows that changed.
- Name the states most likely to regress.
- Keep scenarios concrete enough that QA can act without reading the diff.
