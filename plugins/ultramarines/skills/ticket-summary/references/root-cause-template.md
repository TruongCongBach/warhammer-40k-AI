# Root Cause Template

## Purpose

Use this template to explain the root cause and connect it cleanly to the fix.

## Template

```markdown
Root cause
- Problem origin: ...
- Why it happened: ...
- Confidence: confirmed | likely | inferred

Fix approach
- Strategy used: ...
- Why this addresses the cause: ...
```

## Rules

- Keep the root cause statement specific.
- If the cause is not fully proven, mark the confidence clearly.
- Make the fix approach read as a response to the cause, not as a detached code summary.
