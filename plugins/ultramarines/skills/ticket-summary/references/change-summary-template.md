# Change Summary Template

## Purpose

Use this template for the main delivery summary.

## Template

```markdown
Suggested title
- type: TICKET-ID | short summary

Suggested commit title
- type: TICKET-ID | short summary

Suggested summary title
- type: TICKET-ID | short summary

Suggested QA handoff title
- type: TICKET-ID | short summary

Ticket summary
- Ticket: TICKET-ID
- Type: bug | feature | chore | refactor | hotfix
- Summary: ...

Root cause
- ...

Fix approach
- ...

What changed
- ...

Affected areas
- ...

Regression risks
- ...

QA focus areas
- ...

Suggested test scenarios
- ...

Short summary for ticket history
- ...

Short summary for engineering readers
- ...
```

## Writing Rules

- Prefer short bullets over long prose.
- Keep root cause and fix approach separate.
- Describe impact in terms of behavior and scope.
- Avoid low-signal changelog wording.
